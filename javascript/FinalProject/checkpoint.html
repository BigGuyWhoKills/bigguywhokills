Main components of the 'game' logic:
  1. When categoryItem objects are created, the net value is determined at creation-time. - Implemented
    While this is implemented, it is not a user-observable activity.
  2. When categoryItem objects are pushed onto the array, a totals categoryItem object is created. - Implemented
    While this is implemented, it is not a user-observable activity.
  3. Cell style is only changed if a value has been modified. - Implemented
    Click on a cell, enter a new value in the prompt box, and click OK.
  4. When a cell is clicked, display the row and column number of that cell. - Implemented
    Click on any cell. The values are displayed lower on the page.
    Note that the table header cells are not clickable, but are considered to be a row in the table, so row 0 is not observable. However, column 0 is observable.
  5. Recalculate the cell values after changes. - NOT implemented
    This will require reading in the entire table, parsing each cell to get its new value, and rebuilding the table.
  6. Remove rows. - Partially implemented
    Currently, this is hard-coded to the initial row names.
    I intend to update this to eventually read the first column from each row, and populate the list with that.
  7. Add a new row. - Partially implemented
    The form is set up for this, but the code is not written yet.
